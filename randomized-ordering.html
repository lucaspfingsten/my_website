Randomized Ordering

Randomized ordering protocols gather transactions over a set period and then execute them in a random order to prevent front-running on the blockchain. This approach ensures unpredictability and fairness in transaction processing but may not be ideal for time-sensitive trades.

Positives

  • Good Scope
  • Minimal Jostling

Negatives

  • Increased cost
  • Increased delay
  • Reduced decentralization
  • Security in case of byzantine committee

Methods

BlockSwap

BlockSwap secures transactions using a commit-reveal scheme coupled with randomized ordering. Initially, only the hash of a transaction is submitted to conceal details and prevent front-running. Upon revealing these transactions, their order is randomized through consensus. Thus, no single entity controls the sequence.

Partitioned and Permuted Protocol

The Partitioned and Permuted Protocol (Π3) is a blockchain protocol designed to prevent transaction reordering attacks through randomizing the order of transactions within each block. The protocol uses a uniform random permutation, created by a group of leaders (recent miners in the blockchain), to shuffle the transaction order unpredictably. This method ensures that the relative ordering of transactions is unknown, making MEV attacks less profitable, while maintaining the security and efficiency comparable to existing blockchain protocols.

For more information read the website: 
Eating Sandwiches: Modular and lightweight elimination of transaction reordering attacks

BlindPerm

The BlindPerm protocol aims to mitigate MEV in blockchain networks by combining an encrypted mempool with randomized transaction ordering. Transactions are first encrypted and submitted to the mempool, ensuring privacy. Before block execution, the BlindPerm protocol randomly permutes the order of transactions in a block, making it difficult for malicious actors to predict and exploit transaction ordering for MEV.

For more information read the paper:
BlindPerm: Efficient MEV Mitigation with an Encrypted Mempool and Permutation
