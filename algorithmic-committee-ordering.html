Algorithmic Committee Ordering

In algorithmic committee ordering schemes, a committee oversees the transaction ordering. Transactions are sent directly to the committee and the committee agrees upon a fair ordering through consensus. Generally, these approaches can handle less than one third of the committee members being byzantine.

Positives

  • Goodput stays unchanged
  • No significant delay

Negatives

  • Medium Scope as back-running still possible
  • Bad security as attacks still possible
  • Reduced decentralization
  • Increased costs
  • Jostling increased

Methods

Hashgraph

The Hashgraph protocol is a distributed consensus algorithm. It uses a gossip-about-gossip mechanism, where each node communicates information (gossip) about transactions and other nodes' gossip, creating a Hashgraph. This graph records the history of messages in a network, ensuring that the order of transactions is consistent across all nodes. Precisely, the protocol employs virtual voting to achieve consensus on the order of transaction, i.e.,nodes can  deduce the votes of others.

For more information read their paper:
The Swirlds Hashgraph Consensus Algorithm: Fair, Fast, Byzantine Fault Tolerance

Quick Order-Fair Broadcast Protocol

The Quick Order-Fair Atomic Broadcast Protocol operates by broadcasting messages to all network members while keeping track of the order in which these messages are sent and received. The protocol uses a consensus mechanism to agree on the order of messages, ensuring that all members have a consistent view of the message sequence.

For more information read their paper: 
Quick Order Fairness

Pompe

The Pompe protocol builds on SMR in the context of permissioned blockchains. It introduces a new Byzantine Ordered Consensus primitive to lessen the control Byzantine nodes can have over the order of transactions. The protocol is designed to ensure ordering linearizability, i.e., if the highest timestamp from all correct nodes for transaction t_1 is lower than the lowest timestamp from all correct nodes for transaction t_2, than transaction t_1 executed before transaction t_2.

For more information read the paper:
Byzntine Ordered Consensus without Byzantine Oligarchy

Wendy

The Wendy protocol modifies the transaction ordering process. It operates as an additional layer over standard blockchain protocols, selectively applied to certain transactions. Wendy focuses on relative and probabilistic fairness mitigate intentional reordering or delaying of transactions by network participants.

For more information read the paper:
Wendy, the Goot Little Fairness Widget

Aequitas

The Aequitas protocol ensures order-fairness in blockchain networks through a two-stage process. In the Broadcast Stage, nodes receive transactions and broadcast them using a FIFO method, creating a consistent view of transaction orderings. The Finalization Stage involves no further node communication, as each node computes the final transaction ordering locally from received input orderings. This results in a transaction sequence that reflects their original reception order. There is a protocol for both synchronous and asynchronous settings.

For more information read the paper: 
Order-Fairness for Byzantine Consensus

Themis

Themis operates in a partially synchronous environment. It stands out for implementing the batch-order-fairness property, i.e., it ensures that if transaction t_1 is received before transition t_2 first by a certain fraction of nodes are ordered accordingly. Its design integrates seamlessly into existing leader-based consensus protocols with minimal changes: replicas send transaction order information to the leader, who then constructs a fair block proposal. This is followed by a verification mechanism for replicas to assess the fairness of the proposal and finalize the transaction ordering.

For more information read the paper:
Themis:Fast, Strong Order-Fairness in Byzantine Consensus

Order-Fair Consensus in the Permissionless Setting

The OrdeFair Consensus in the Permissionless Setting paper effectively combats transaction reordering. It employs a soft-ordering mechanism, which ensures that transactions are processed in the order they are received by the network, provided they arrive with a significant time difference. This approach is crucial in permissionless blockchains where no single entity has control over the network, significantly reducing the risk of reordering attacks commonly exploited by miners or influential network participants.

For more information read the paper:
Order-Fair Consensus in the Permissionless Setting

DCN

The DCN protocol achieves transaction order by using a network of independent clocks. These clocks, operating under an agreement protocol, assign timestamps to transactions. By uniformly timestamping transactions across the network, DCN creates a more equitable and consistent transaction ordering process.

For more information read the paper:
A Fair and Resilient Decentralized Clock Network for Transaction Ordering
